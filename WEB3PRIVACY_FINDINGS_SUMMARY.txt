================================================================================
                         SCORING METHODOLOGY
================================================================================

PROPOSED CONFIDENCE SCORING (0.0-1.0):
  1.0  = Official source, HTTP 200 verified, recent update
  0.9  = Official source verified, data < 1 week old
  0.8  = Official OR 2+ independent sources verified
  0.7  = One official + one secondary source verified
  0.6  = Secondary sources only, cross-verified
  0.5  = Single source, older data
  <0.5 = Unverified, conflicting sources, or placeholder

COMPLETENESS SCORING (0-100%):
  = (populated_fields / important_fields) * 100
  Important fields: name, description, links, category
  Our average: 58% (23 projects at 70%+)

MULTI-SOURCE VERIFICATION:
  Required minimum: 1 source per field
  Target: 2+ sources for high-confidence data
  Includes: website, GitHub API, documentation, team sources
  All with HTTP status codes and retrieval timestamps

FRESHNESS METRIC (0.0-1.0):
  1.0  = Updated within 1 day
  0.9  = Updated within 1 week
  0.8  = Updated within 1 month
  0.5  = Updated 3-6 months ago
  <0.5 = Last update > 6 months ago

MISSING FIELDS REPORTING:
  Transparent array of unfound data
  Examples: "token_economics", "recent_news", "detailed_roadmap"
  Helps community identify gaps they can fill

                      PROPOSED ENHANCEMENT SCHEMA
================================================================================

Add to Web3Privacy YAML (NEW OPTIONAL SECTION):

data_quality:
  confidence: 0.95                    # Overall 0.0-1.0
  completeness: 0.80                  # Percentage 0.0-1.0
  accuracy: 0.95                      # Field accuracy confidence
  freshness: 0.85                     # How recent (0.0-1.0)
  verification_date: "2025-10-10"     # ISO 8601
  verification_count: 2               # Number of sources
  
  sources:
    - type: website
      url: https://aztec.network
      verified: true
      http_status: 200
      retrieved_at: "2025-10-10T14:30:00Z"
    - type: github-api
      url: https://api.github.com/repos/AztecProtocol
      verified: true
      retrieved_at: "2025-10-10T14:31:00Z"
  
  missing_fields:
    - token_economics
    - recent_news
    - detailed_roadmap
  
  notes: "Verified from official website, GitHub API, and documentation"

BENEFITS:
- Backward compatible (optional section, doesn't break existing data)
- Transparent about data quality
- Enables filtering/sorting by quality
- Community knows what's missing
- Cross-validates multiple sources
